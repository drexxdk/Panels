Introduction

  • The goal with this website is to make a dynamic layout with all the features a modern website should have, that works in all browsers on all devices.
  • Originally the website used Bootstrap and FontAwesome, but it bloated the website and effected performance, therefore I removed them after copying the parts from Bootstrap I needed and added it to my own SCSS files.
  • FontAwesome made it easy to add icons for everything, but it was the biggest resource that had to be downloaded at page load, therefore I replaced it with svg icons, that got added directly to the DOM.

Some of the features

  • Keyboard controls is disabled when loading is shown.
    • Left arrow key if modal isn't shown it will opens settings or closes navigation if it is open.
    • Right arrow key if modal isn't shown it will opens navigation or closes settings if it is open.
    • ESC key closes modal or aside, popups and dropdowns.
    • TAB toggles modal, aside, dropdown if focused element is within/outside of it.
    • ENTER toggles checkbox, radio, dropdown, accordion item, navigation tree item if it has focus.
  • Local storage is used to save navigation and settings.
  • Modal for image keeps aspect ratio and always fills entire width or height of screen.
  • Tables can be flipped without any javascript.
  • REM is used for everything instead of EM or PX. REM gets converted to PX for breakpoints, since browsers can't handle REM in breakpoints correctly.
  • Media queries based on layout elements instead of just browser window width.
  • SVG icons with fill color that can be set in SCSS.
  • Hover effects is only applied for desktop browsers, since mobiles can't display them correctly.
  • Responsive design that works down to around 300x300px.
  • Focus is applied to all inputs for accessibility.
  • Scroll is disabled for the site, when displaying modal, aside without shrink and loading.
  • Transitions works in all browsers by using margin/padding alternatives to calc(), since Edge/IE can't handle transitions with calc().
  • Android
    • Modal image shrinks/grows, when androids top bar is shown/hidden.
    • Swipe controls is disabled when loading or modal is shown or when horizontally scrolling.
      • Swipe left opens settings or closes navigation if it is open.
      • Swipe right opens navigation or closes settings if it is open.
  • Visual Studio
    • Web Compiler by Mads Kristensen is used to convert SCSS to CSS with sourcemap and minified version.
    • Bundler & Minifier by Mads Kristensen is used to combine and minify JS.
  • SPA by Rafael Pedicini is used to make the website into a Single-Page-Application.
  • loadCSS.js by Filament Group is used to load CSS.
  • lazy.js by Eisbehr is used to load images.
  • bowser.js by Dustin Diaz is used to determinate user browser and device.
  • tooltipster.js by Caleb Jacob is used to make tooltips.
  • katex.js by Khan Academy is used to display mathematics.
  • jquery.validate.js by Jörn Zaefferer is used for form validation.
  • sortable.js by Lebedev Konstantin is used for drag and drop.

Settings

Settings work by applying classes to html and saving it in local storage.

  • Fixed footer
    • Enabled: Footer is fixed at bottom of screen.
    • Disabled: Footer is shown below content.
  • Two columns is disabled untill screen is wide enough.
    • Enabled: Content is displayed in two columns.
    • Disabled: Content is displayed in one column.
  • Transitions
    • Enabled: Layout uses transitions.
    • Disabled: Layout doesn't use transitions.
  • Shrink is disabled untill screen is wide enough.
    • Enabled: Aside shrinkes content.
    • Disabled: Aside opens on top of content.
  • Close on click outside is always enabled when aside isn't shrinking content.
    • Enabled: Close aside when clicking outside it, unless target is loading, modal or popup.
    • Disabled: Don't close aside when clicking outside it.
