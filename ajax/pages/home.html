Introduction

  • The goal with this website is to make a dynamic layout, with all the features a modern website should have, that works in all browsers and on all devices.
  • Inspired by Bootstrap and Foundation.

Some of the features

  • Keyboard controls is disabled when loading is shown.
    • Left arrow key if focus and modal isn't shown it opens settings or closes navigation if it is open.
    • Right arrow key if focus and modal isn't shown it opens navigation or closes settings if it is open.
    • ESC key closes focus, modal or aside, authentication, popups and dropdowns.
    • TAB toggles authentication, modal, aside, dropdown if focused element is within/outside of it.
    • ENTER toggles authentication, checkbox, radio, dropdown, accordion item, navigation tree item if it has focus.
  • Local storage is used to save navigation and settings.
  • Modal for image keeps aspect ratio and always fills entire width or height of screen.
  • Tables can be flipped without any javascript.
  • REM is used for everything instead of EM or PX. REM gets converted to PX for breakpoints, since browsers can't handle REM in breakpoints correctly.
  • Media queries based on layout elements instead of just browser window width.
  • SVG icons with fill color that can be set in SCSS.
  • Hover effects is only applied for desktop browsers, since mobiles can't display them correctly.
  • Responsive design that works down to around 300x300px.
  • Focus is applied to all inputs for accessibility.
  • Scroll is disabled for the site, when displaying modal, aside without shrink and loading.
  • Transitions works in all browsers by using margin/padding alternatives to calc(), since Edge/IE can't handle transitions with calc().
  • Android
    • Modal image shrinks/grows, when androids top bar is shown/hidden.
    • Swipe controls is disabled when loading or modal is shown or when horizontally scrolling.
      • Swipe left opens settings or closes navigation if it is open.
      • Swipe right opens navigation or closes settings if it is open.

Settings

Settings work by applying classes to html and saving it in local storage.

  • Fixed footer
    • Enabled: Footer is fixed at bottom of screen.
    • Disabled: Footer is shown below content.
  • Two columns is disabled untill screen is wide enough.
    • Enabled: Content is displayed in two columns.
    • Disabled: Content is displayed in one column.
  • Transitions
    • Enabled: Layout uses transitions.
    • Disabled: Layout doesn't use transitions.
  • Signed in
    • Enabled: Displays "User" button in header.
    • Disabled: Displays "Register" and "Login" buttons in header.
  • Android swipe
    • Enabled: Aside can be opened/closed by swiping horizontally on the screen.
    • Disabled: Aside can't be opened/closed by swiping horizontally on the screen.
  • Focus
    • Enabled: Displays "Focus" button in header, that when clicked shows a focus overlay that makes it easier on reading text on the page
    • Disabled: Hides "Focus" button in header
  • Text-to-speech
    • Enabled: Displays "Text to speech" toggle in header, that when enabled plays the text the user selects
    • Disabled: Hides "Text to speech" button in header
  • Theme can be switched between light and dark. Only the selected theme will be loaded at initial page load, before the page gets displayed.
  • Aside
    • Overlay: Aside opens on top of content.
    • Push: Aside pushes content.
    • Shrink: Aside shrinks content. (uses "Overlay" untill screen is wide enough)
  • Close on click outside is always enabled when aside isn't shrinking content.
    • Enabled: Close aside when clicking outside it, unless target is loading, modal or popup.
    • Disabled: Don't close aside when clicking outside it.
  • Close on page change
    • Enabled: Close aside when changing page
    • Disabled: Don't close aside when changing page.
