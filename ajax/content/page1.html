Introduction

  • The goal of this site is to get a good score at Google Lighthouse, but still have the features I want.
  • Originally the site used Bootstrap and FontAwesome, but it bloated the site and effected performance, therefore I removed them after copying the parts from Bootstrap I needed and added it to my own SCSS files.
  • FontAwesome made it easy to add icons for everything, but it was the biggest resource that had to be downloaded at page load, therefore I replaced it with svg icons, that got added directly to the DOM.

Some of the features

  • Keyboard controls:
    • Left arrow key opens settings or closes navigation if it is open.
    • Right arrow key opens navigation or closes settings if it is open.
    • ESC key closes aside, popups and fullscreen.
  • SCSS to CSS with sourcemap.
  • CSS loaded with loadCSS.js.
  • JS combined and monified.
  • ServiceWorker.
  • Media queries based on layout elements instead of just browser window width.
  • SVG icons with fill color that can be set in SCSS.
  • Hover effects are only applied for desktop browsers, since mobiles can't display them correctly.
  • #content images are loaded with lazyload.
  • Position fixed asides that works correctly in all browsers including Safari mobile that usually breaks position fixed elements.
  • Scroll for the site will be disabled when displaying fullscreen image, aside without push and loading.

Bugs

  • Chrome Desktop/IE cannot display rem values correctly. This causes some components like 'switch' to be displayed wrong at some zoom levels.
  • Edge/IE cannot display multiple transitions simultaneously correctly. Transitions are therefore disabled for the layout in those browsers.
