Introduction

  • The goal of this site is to get a good score at Google Lighthouse, but still have the features I want.
  • Originally the site used Bootstrap and FontAwesome, but it bloated the site and effected performance, therefore I removed them after copying the parts from Bootstrap I needed and added it to my own SCSS files.
  • FontAwesome made it easy to add icons for everything, but it was the biggest resource that had to be downloaded at page load, therefore I replaced it with svg icons, that got added directly to the DOM.

Some of the features

  • Keyboard controls
    • Left arrow key opens settings or closes navigation if it is open.
    • Right arrow key opens navigation or closes settings if it is open.
    • ESC key closes aside, popups and fullscreen.
  • Swipe controls
    • Swipe left opens settings or closes navigation if it is open.
    • Swipe right opens navigation or closes settings if it is open.
  • Local storage is used to save settings.
  • Fullscreen keeps aspect ratio and always fills entire width or height of screen.
  • Tables can be flipped without any javascript.
  • REM is used for everything instead of EM or PX. REM gets converted to PX for breakpoints, since browsers can't handle REM in breakpoints correctly.
  • ServiceWorker is used for to make it possible to view the site offline and caching.
  • Media queries based on layout elements instead of just browser window width.
  • SVG icons with fill color that can be set in SCSS.
  • Hover effects are only applied for desktop browsers, since mobiles can't display them correctly.
  • Scroll is disabled for the site, when displaying fullscreen image, aside without push and loading.
  • Transitions works in all browsers by using margin/padding alternatives to calc(), since Edge/IE can't handle transitions with calc().
  • Android
    • Swipe down will refresh page by default. That has been disabled, since it often happens by accident.
    • Fullscreen shrinks/grows, when androids top bar is shown/hidden.
  • Visual Studio
    • Web Compiler by Mads Kristensen is used to convert SCSS to CSS with sourcemap and minified version.
    • Bundler & Minifier by Mads Kristensen is used to combine and minify JS.
  • loadCSS.js by Filament Group is used to load CSS.
  • lazyload.js by Mika Tuupola is used to load content images.
  • bowser.js by Dustin Diaz is used to determinate user browser and device.
  • bootstrap-slider.js by Stefan Petre is used to make sliders.

Settings

Settings work by applying classes to html and saving it in local storage.

  • Fixed footer
    • Enabled: Footer is fixed at bottom of screen. Disabled for mobile.
    • Disabled: Footer is shown below content.
  • Two columns
    • Enabled: Content is displayed in two columns.
      Content is displayed in one column untill the screen is wide enough to switch to two columns.
    • Disabled: Content is displayed in one column.
  • Transitions
    • Enabled:Layout uses transitions for the layout.
      Disabled for Edge and IE since they can't display transitions for elements that uses calc().
    • Disabled: Layout doesn't use transitions for the layout.
  • Push
    • Enabled: Aside pushes content.
      Aside opens on top of content untill the screen is wide enough for it to push content.
    • Disabled: Aside opens on top of content.
  • Close on click outside
    • Enabled: Close aside when clicking outside it, unless target is #loading, #fullscreen or .popup.
      Always enabled when aside isn't pushing content.
    • Disabled: Don't close aside when clicking outside it.

Bugs

  • Edge/IE doesn't support object-fit. Cover images are therefore made with background-image, but that can't load image based on browser window width, unless it is done with javascript.
    Have tried using polyfills, but can't get them to work.
    This increases page load time and forces mobile users to download larger images than their screen can show.
